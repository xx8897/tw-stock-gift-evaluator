# 台股股東會紀念品攻略與評估工具 🎁

這是一個專為台灣股民（特別是**零股股東**）打造的股東會紀念品攻略庫。內含一套能自動抓取最新股價並計算「零股性價比 (CP值)」的 Python 評估腳本，以及一個精心設計的互動式網頁前端。

🌐 **主力站點 (Vercel)**：[https://tw-stock-gift-evaluator.vercel.app](https://tw-stock-gift-evaluator.vercel.app)
🔗 **備用網址 (GitHub Pages)**：[https://xx8897.github.io/tw-stock-gift-evaluator/](https://xx8897.github.io/tw-stock-gift-evaluator/)

## 🧶 策略理念：零股小資撿禮，把小錢放大

> **花幾百元買進 1 股零股，每年參加股東常會，嚕一點紀念品，同時還能收一點股息。**

台灣有許多股票，**單股股價在幾元到幾十元之間**，持有 1 股零股的門檻極低。只要在公司股東會紀念品的「基準日」（通常是除息/除權前）持有至少 1 股，就有資格以零股股東身分請人代領股東會紀念品。

### 為什麼值得做？
- 💸 **成本極低**：許多高 CP 值標的股價不到百元，進場完全無壓力。
- 🎁 **實質回饋**：從超商商品卡、日常用品、到六福村遊樂園門票（市值 1,199 元），有些標的光靠紀念品就能「回本」不只一次。
- 🔁 **年年可領**：只要公司持續發放，每年都能拿，最理想的標的是「五年連續發放」的穩定大戶。
- 🧮 **科學化精算**：本工具透過獨家公式 `（五年紀念品總估值 ÷ 股價）× （發放次數 ÷ 5）`，一眼就能看出哪支是真正超值的神股。

> **⚠️ 免責聲明**：本工具純粹以「紀念品 CP 值」作為排序依據，不構成任何投資建議。任何股票的基本面、業績、流動性等因素請自行評估。

  - 專案本身不使用任意字串 `eval` 類型執行，降低潛在 XSS 風險。

- **表單與使用者體驗**
  - 所有表單欄位皆搭配對應的 `label` 與適當的 `autocomplete` 設定，改善自動填寫體驗與可及性。
  - 登入、註冊、OTP 驗證、忘記密碼、暱稱修改與意見回饋等流程，皆具備清楚的 loading / error / success 狀態。

- **安全性與隱私 (Security & Privacy)**
  - **Google 安全授權**：採用 Google OAuth 2.0 驗證技術，登入過程直接由 Google 伺服器處理，本站**不會且無法**取得您的密碼，確保帳號安全。
  - **極簡權限請求**：僅要求 E-mail 作為同步識別碼，不請求聯絡人、檔案或其他敏感權限，尊重使用者隱私。
  - **資料傳輸保護**：資料儲存於企業級安全性相容的 **Supabase** 雲端，且所有通訊皆全程透過 HTTPS 加密傳輸，請放心使用。

- **品質與除錯體驗**
  - 前端在 Chrome DevTools 的 Console 與 Issues 面板維持 **0 error**，避免隱性 runtime 問題。
  - 透過模組化拆分（`data`, `table`, `ui`, `auth`, `ranking` 等），搭配 Source Map，讓日後維護與擴充更容易。

## 🚀 關於估值引擎 `valuation_v5.py`

這是本專案的資料心臟，驅動全台 1,600+ 支股票的紀念品 CP 值計算。引擎讀取 `gift_tree/tree.json` 這棵「知識分類樹」的規則，透過 DFS 演算法尋找最精確的分類，再繼承節點的屬性進行估算。

### 📊 典型品項估值範例

| 品名範例 | 分類節點 | 底價 | 倍數 | 代領成本 | **最終估值** |
|---------|---------|-----:|-----:|--------:|------------:|
| 六福村主題樂園門票 | 🎫 票券與高價特賞 > 六福村 | 799 元 | × 0.7 | − 15 | **544** 元 |
| 大魯閣娛樂抵用券 | 🎫 票券與高價特賞 > 大魯閣 | 800 元 | × 0.7 | − 15 | **545** 元 |
| 王品股東大禮包 | 🎫 票券與高價特賞 > 王品 | 2,800 元 | × 0.7 | − 15 | **1,945** 元 |
| 全家超商禮物卡 35 元 | 💰 禮物卡 > 超商 > 35 | 35 元 | × 1.0 | − 15 | **20** 元 |
| 全家超商大杯拿鐵 | 💰 禮物卡 > 超商 > 55咖啡 | 55 元 | × 1.0 | − 15 | **40** 元 |
| 不鏽鋼燉煮鍋 | 🍳 寢具與大型烹飪器具 > 大型燉煮鍋 | 300 元 | × 1.0 | − 20 | **280** 元 |
| 行動電源 | ⚡ 3C 與家電 > 個人電子 | 300 元 | × 1.0 | − 20 | **280** 元 |
| （無法分類物品） | — | 40 元 | × 1.0 | − 20 | **20** 元 |

> 💡 所有規則皆儲存於 `gift_tree/tree.json`，可透過視覺化編輯器 `gift_tree/app.py` 隨時調整，無需修改程式碼。

## 🧠 V5 知識分類樹估值引擎——設計哲學

> *「與其把所有規則寫死在程式碼裡，不如把規則變成可以編輯的資料。」*

### 問題的根源：硬編碼的地獄

在 V4 以前，估值邏輯是一坨龐大的 if-else 巢狀結構：

```python
if '大魯閣' in gift: val = 800
elif '六福村' in gift: val = 1199
elif '不鏽鋼' in gift and '杯' in gift: val = 200
# ... 繼續綿延超過 60 行 ...
```

每次要新增一個贈品類型，就得打開 Python 程式碼、搞清楚順序邏輯、然後祈禱不會 Regression。**規則不透明、不可視、無法被非工程師修改。**

### V5 的天才突破：把「規則」與「程式」分離

V5 引入了一套以 **`tree.json`** 為核心的**知識分類樹 (Taxonomy Tree)**，所有的分類智慧都儲存在一棵可視化的樹狀 JSON 結構中，再由輕量的 Python 引擎讀取並執行。兩者徹底解耦：

```
┌────────────────────────────────────┐
│  gift_tree/tree.json               │  ← 「大腦」：可視化編輯的知識圖譜
│  (人類可讀的 JSON 規則定義)         │
└──────────────┬─────────────────────┘
               │ DFS 搜尋
               ▼
┌────────────────────────────────────┐
│  scripts/core/valuation_v5.py      │  ← 「執行器」：純粹的邏輯引擎
│  (不含任何品項知識的通用演算法)     │
└──────────────┬─────────────────────┘
               │
               ▼
        最終估算價格 💰
```

### 三大設計亮點

#### 1. 深度優先搜尋 (DFS) 精準比對
引擎對每件贈品執行 DFS，在樹狀結構中找到**最深層的匹配節點**。每個節點可設定：
- `keywords`（OR 匹配）、`keywords_and`（AND 全符合）、`keywords_not`（排除）

比如「統一超商大杯拿鐵」這個品名，會精準落到 `超商 > 55咖啡` 這個節點，對應 base_value=55，而不是只匹配到泛用的「超商」節點（base_value=30）。**越精確的描述，越精準的估值。**

#### 2. 屬性向上繼承 (Attribute Inheritance)
找到最深節點後，引擎往上回溯，繼承第一個有設定的：
- `base_value`（基礎底價）
- `value_multiplier`（價值倍數）

這讓「票券與高價特賞」的 `value_multiplier: 0.7` 可以自動套用到所有子節點（六福村、王品、大魯閣等），**無需為每個子節點個別設定**。

#### 3. 標籤驅動的成本計算 (Tag-Driven Cost Engine)
引擎同樣會往上回溯收集所有祖先節點的 `tags`，再依據標籤決定代領成本：

| 標籤 | 代領成本 | 說明 |
|------|---------|------|
| `票券` | 15 元 | 票券類（門票、餐券等），需一定代領門檻 |
| （無） | 20 元 | 一般實體品項，寄送/代領標準成本 |

這意味著，**新增一種物品類型只需要在 `tree.json` 的對應節點打上 tag**，全部的計算邏輯自動生效。

### 可視化的樹狀編輯器

更妙的是，所有這一切都透過本地端的視覺化編輯器 (`gift_tree/app.py`) 管理：

```bash
python gift_tree/app.py
# 打開 http://127.0.0.1:5001
```

非工程師也能透過瀏覽器拖曳、新增、修改分類節點與關鍵字，修改後**全庫 1,600+ 支股票的 CP 值即可一鍵重新試算**，不需要動任何一行程式碼。
