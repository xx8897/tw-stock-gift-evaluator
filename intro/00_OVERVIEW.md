# 這不是另一個股票篩選器

> 買 1 股 2330 台積電要 950 元，紀念品是一把螺絲起子。買 1 股 0056 元大高股息要 22 元，紀念品是 200 元禮物卡。差別在哪？差別在一個數字：CP+。

## 一句話

**synthforge** 是規則框架，**ETF AI Advisor** 是 AI 推薦引擎，**這個專案是一個真的有人用的產品**——幫台灣散戶計算「買 1 股零股 + 領紀念品」的投資回報率。

## 解決什麼問題

台灣上市公司每年開股東會，很多公司會發紀念品。散戶只要買 1 股零股（通常幾十到幾百塊），就能出席或委託代領。

問題是：
- 紀念品值多少？沒有人系統性地估價過
- 哪一家最划算？需要同時考慮股價、紀念品價值、發放頻率、五年紀錄
- 什麼時候該買？最後買進日是哪天？
- 領取會不會有風險？需不需要身分證件？

這個工具回答了所有這些問題。

## 為什麼這個專案值得看

這個專案值得看的不是功能列表——功能列表任何股票網站都有。值得看的是三件事：

### 1. 知識樹引擎

從 500 行 if-else 到 200 行 DFS，中間發生了什麼？這是這個專案最核心的故事。V4 的估價邏輯是一條條硬編碼的判斷，每季都要改代碼。V5 把所有商品知識搬到了 JSON 樹裡，代碼只做搜尋和計算，不包含任何商品知識。

**這不是小重構，是範式轉移。** 詳見 [01_IF_ELSE_TO_TREE.md](01_IF_ELSE_TO_TREE.md)。

### 2. 一個真的有人用的產品

GitHub Pages 靜態部署 + Supabase BaaS + GitHub Actions 定時更新 = 每月成本接近零。沒有 Kubernetes，沒有微服務，沒有 Docker。但是有人真的每天在用它決定要買哪支股票。

### 3. 建造者的 hack

金絲雀檢查（用台積電測試是不是交易日）、15 分鐘 debounce 同步、7.2 秒強制載入動畫、匿名 UUID 追蹤、正則表達式捕捉禮物卡面額——這些小東西是在真實使用中長出來的，不是設計出來的。

## 幾個關鍵數字

- **1,680 行** tree.json 禮物分類樹
- **9 大類別** + 1 個「雜物」桶
- **200 行** valuation_v5.py（V4 是 500+ 行 if-else）
- **5 個分隔符** 處理複合禮物（+, &, 與, 和, 及）
- **5 星硬門檻** 不是連續分數，是閘門決策
- **0 個後端伺服器** 面向使用者
- **200 支** 每次更新輪流的股票數量
