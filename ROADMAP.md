## 🚧 進行中 / 近期計劃

### 「最後買進日」欄位
從 MOPS 公告的「停止過戶日」自動反推最後買進日，在前端顯示倒數提醒，協助使用者掌握進場時機。

### 全量 CP 值重算流程優化
目前 `evaluate_stocks_supa.py` 每次需手動執行，考慮整合進 GitHub Actions 排程，做到「股價一更新，CP 值自動跟著重算」的全自動循環。

### `voting_status` 即時投票現況表
計劃新增一張資料表，持續追蹤**目前正在開放電子投票且會發紀念品**的股票清單，讓使用者更清楚「現在該上哪台車」。

| 欄位 | 說明 |
|---|---|
| `stock_id` | 股票代號 |
| `last_buy_date` | 最後買進日（停止過戶前 2 個交易日） |
| `meeting_date` | 股東會日期 |
| `souvenir` | 本次紀念品品項 |
| `zero_share_eligible` | 零股是否可領（布林值） |

## 🧱 技術架構定案

| 面向 | 決策 | 說明 |
|---|---|---|
| 資料庫 | Supabase PostgreSQL | 公開股票資料用公開讀取，個人清單用 RLS 保護 |
| 前端狀態 | `AppState` 全域物件 | 所有過濾狀態、分頁、排序集中管理 |
| 估值引擎 | `valuation_v5.py` + `tree.json` | 規則與演算法徹底解耦，規則改動不觸碰程式碼 |
| CI/CD | GitHub Actions | 每週一、三、五台灣時間 18:00 執行，爬取 FinMind 後直存 Supabase |
| 同步策略 | 防抖 + 15 分鐘定時 | 避免每次操作都打 API，降低雲端消耗 |
