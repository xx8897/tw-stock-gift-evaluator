# FinMind 股價分批更新執行計畫 (每小時 200 支)

為了在滿足 FinMind API 免費版/基礎版頻率限制（每小時 300-600 次）的前提下，穩定更新專案中 800+ 支股票的股價，我們擬定以下執行計畫。

## 1. 核心策略：增量更新 (Incremental Updates)
腳本將不再每次執行都嘗試更新「所有」股票，而是改為：
1.  **檢查日期**：讀取 Excel，篩選出「最近股價日期」早於「今日（或最新交易日）」的股票。
2.  **限量抓取**：從篩選出的名單中，僅取前 **200 支** 進行 API 請求。
3.  **自動填補**：下次執行時，前 200 支已更新完成，腳本會自動往下抓取後續的 200 支。

## 2. GitHub Actions 調整
將 `.github/workflows/update-prices.yml` 的觸發頻率從每日一次改為 **每小時一次 (Hourly)**。

*   **預期進度**：
    *   T+0 小時：更新 1~200 支。
    *   T+1 小時：更新 201~400 支。
    *   T+2 小時：更新 401~600 支。
    *   T+3 小時：更新 601~800 支。
    *   T+4 小時：更新剩餘部分，全市場更新完成。

## 3. 實作細節 (update_prices_v2.py)
*   **API 認證**：使用 `FINMIND_TOKEN` 確保請求成功率。
*   **延遲保護**：每次 API 請求間隔 0.5 秒，避免觸發伺服器瞬間防禦。
*   **存檔機制**：每次執行完成後，立即將進度存回 Excel 並 Push 至 GitHub，確保資料連續性。

## 4. 效益評估
*   **穩定性**：請求次數遠低於每小時 600 次的上限，極度穩定。
*   **容錯性**：若某次執行失敗，下一小時會自動補跑，不影響整體進度。
