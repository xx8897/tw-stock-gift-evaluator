# 星級評等的硬門檻

> 一支 CP+ 5.0 的股票，如果五年只發過一次紀念品，它只有 2 星。這不是 bug，是刻意設計。

## 為什麼不是連續分數

最直覺的評等方式是加權平均：

```
score = 0.4 × cp_normalized + 0.3 × freq_normalized + 0.3 × risk_normalized
```

問題是，加權平均會掩蓋致命風險。

假設有一支股票：
- CP+：5.0（超高性價比！）
- 頻率：5 年 1 次（只發過一次）
- 風險：高門檻（需要 1,000 股才能領取）

加權平均可能算出 3.5 分——看起來還不錯。但實際上，一支五年只發過一次、且需要大額持股才能領取的股票，風險極高。3.5 分給了錯誤的安全感。

星級系統用閘門邏輯代替加權平均。一個閘門不通過，直接降級：

```
5 星：所有閘門通過
  ├── CP+ ≥ 2.0？            → 否 → 降到 4 星
  └── 頻率 ≥ 4/5？           → 否 → 降到 3 星

4 星：高性價比，但不完美
  └── CP+ ≥ 1.0？            → 否 → 降到 3 星

3 星：基本合格但有風險
  └── 頻率 ≥ 3/5？           → 否 → 降到 2 星

2 星：明確風險
  └── 無額外閘門（已經是低評級）

1 星：高風險
  └── 有零股限制或其他重大風險

0 星：估值為零或數據不足
  └── base_value = 0（雜物桶）或無五年紀錄
```

## 閘門的設計原理

### CP+ 閘門

CP+（性價比）= `五年紀念品總值 / 股價 / 5`

CP+ ≥ 2.0 意味著：每年花 1 元買股票，能回收 2 元以上的紀念品價值。這是一個非常高的門檻——5 星股票的 CP+ 至少是 2.0，意味著紀念品價值至少是股價的兩倍。

CP+ ≥ 1.0 意味著：每年花 1 元，回收 1 元。這是 4 星的最低門檻——至少回本。

### 頻率閘門

頻率是過去五年中發放紀念品的次數。

頻率 ≥ 4/5 意味著：五年中至少發了四次。這是 5 星的門檻——穩定發放。

頻率 ≥ 3/5 意味著：五年中至少發了三次。這是 3 星的門檻——大部分年份有發。

為什麼頻率這麼重要？因為過去的發放頻率是未來發放的最佳預測指標。一支五年只發過一次的公司，明年發放的概率遠低於一支五年發了五次的公司。

### 風險閘門

```
零股限制：需要持有 ≥ 100 股才能領取 → 降 1 星
高門檻：需要親自到場，不接受委託 → 降 1 星
低頻率：頻率 < 2/5 → 降 1 星
估值為零：無法估價（雜物桶）→ 直接到 0 星
```

這些是硬閘門，不是軟扣分。有零股限制就是降級，不是「扣一點分數」。

## 實際例子

### 5 星範例

```
股票：XX 公司
CP+：2.8
頻率：5/5
風險：無

閘門檢查：
  CP+ 2.8 ≥ 2.0 ✓
  頻率 5/5 ≥ 4/5 ✓
  無風險 ✓

結果：★★★★★
```

### CP+ 極高但頻率低

```
股票：YY 公司
CP+：5.0
頻率：1/5
風險：無

閘門檢查：
  CP+ 5.0 ≥ 2.0 ✓
  頻率 1/5 < 4/5 ✗ → 降到最高 3 星

結果：★★☆（即使 CP+ 極高，頻率閘門不通過）
```

### 零股限制

```
股票：ZZ 公司
CP+：1.5
頻率：4/5
風險：零股限制（需 ≥ 100 股）

閘門檢查：
  CP+ 1.5 ≥ 2.0 ✗ → 降到最高 4 星
  頻率 4/5 ≥ 4/5 ✓（但已經因為 CP+ 降到 4 星上限）
  零股限制 → 再降 1 星

結果：★★★（4 星上限 - 1 星風險降級）
```

## 為什麼閘門比加權平均更誠實

閘門邏輯的核心思想是：**有些風險是不可補償的。**

一支 CP+ 5.0 的股票如果五年只發過一次，沒有任何高 CP+ 可以彌補這個不確定性。你看到的 5.0 是過去的數字，明年的頻率可能是 0。

閘門邏輯告訴使用者：「不管 CP+ 多高，低頻率就是低頻率。」這是加權平均做不到的——它會用高 CP+ 去補償低頻率，給出一個看起來合理但實際危險的中間分數。

**零星的誠實**：估值為 0 的紀念品直接給 0 星。不是因為它不值錢，而是因為我們不知道它值多少。誠實的不知道，比虛假的估算更有幫助。

## 星級和排名的關係

在歷史模式，排序是按 CP+；在年度模式，排序是按星級。為什麼不同？

因為歷史模式的讀者是在做長期投資決策，CP+ 是最重要的指標——每年能回收多少價值。年度模式的讀者是在做短期買入決策，星級是綜合指標——考慮了所有風險因素後的整體評價。

兩個排序邏輯服務兩種完全不同的決策場景。
