## 💡 這是什麼？

這是一個專為**台灣零股股東**打造的紀念品 CP 值資料庫與評估工具。

台灣有許多股票，單股股價不到百元，只要在股東會紀念品「基準日」持有至少 1 股，每年就能以零股股東身分請人代領紀念品。

本工具做了兩件事：
1. **後端**：透過 Python 腳本自動抓取最新股價，用 V5 知識分類樹引擎計算每支股票的「性價比+（CP 值）」，並定期同步至 Supabase 雲端資料庫。
2. **前端**：提供互動式網頁，讓使用者篩選、追蹤、比較 800+ 支股票的紀念品品質與 CP 值。

> **⚠️ 免責聲明**：本工具以「紀念品 CP 值」為唯一排序依據，**不構成任何投資建議**。請自行評估標的的基本面、流動性等風險。

## 🧠 V5 估值引擎

> *「與其把所有規則寫死在程式碼裡，不如把規則變成可以編輯的資料。」*

V5 引擎的核心設計是**將「規則知識」與「計算邏輯」完全分離**。

所有的分類知識（哪種紀念品值多少錢）存放在 `gift_tree/tree.json`，這是一棵可以視覺化編輯的 JSON 分類樹。Python 引擎（`valuation_v5.py`）只負責讀取並跑演算法，完全不含任何硬編碼的品項知識。

```
gift_tree/tree.json          ←  「大腦」可視化 JSON 規則
        │
        │ DFS 深度優先搜尋
        ▼
scripts/core/valuation_v5.py ←  「執行器」通用演算法引擎
        │
        ▼
     CP 值 💰
```

### ⚙️ 輕量化 ETL 數據管線

V5 引擎與外部資料庫的互動，本質上構成了一個標準的 **ETL (Extract, Transform, Load)** 流程，透過 GitHub Actions 達成批次處理：

- **🟢 Extract (提取)**：從 FinMind API 抓取最新股價，並讀取本地整理的「紀念品歷史發放紀錄」。
- **🟡 Transform (轉換)**：執行 V5 核心邏輯 `(五年總估值 ÷ 最近價格) × (發放次數 ÷ 5)`，處理異常值並將結果轉換為 1~5 星的推薦等級。
- **🔵 Load (載入)**：將計算完畢的新鮮數據（價格、CP 值、推薦分數）透過 API `UPDATE` 回 Supabase 雲端資料庫供前端即時讀取。

### 估值三步驟

1. **DFS 比對**：用 `keywords`、`keywords_and`、`keywords_not` 在樹中找到最深層的匹配節點。越精確的描述，落點越深，估值越準。
2. **屬性繼承**：從匹配節點往上回溯，繼承第一個設定好的 `base_value`（底價）與 `value_multiplier`（倍數）。
3. **Tag 成本**：回溯所有祖先節點的 `tags`，決定代領成本（票券類 −15 元 / 實體品 −20 元）。

### 典型估值範例

| 品名 | 落點節點 | 底價 | 倍數 | 代領成本 | **最終估值** |
|------|---------|-----:|-----:|--------:|---:|
| 六福村主題樂園門票 | 票券 > 六福村 | 799 | ×0.7 | −15 | **544** |
| 王品股東大禮包 | 票券 > 王品 | 2,800 | ×0.7 | −15 | **1,945** |
| 全家超商禮物卡 35 元 | 超商 > 35 | 35 | ×1.0 | −15 | **20** |
| 不鏽鋼燉煮鍋 | 大型烹飪器具 | 300 | ×1.0 | −20 | **280** |
| （無法分類） | — | 40 | ×1.0 | −20 | **20** |

## 🛠 工程品質

- **Content Security Policy (CSP)**：嚴格限制腳本與連線來源，防止 XSS。支援 Cloudflare Turnstile、Google Analytics、Supabase 等必要服務。
- **流量與行為分析**：整合 **Google Analytics 4 (GA4)** 進行基礎流量追蹤與自訂自訂點擊事件（Click Events）蒐集，作為精準產生「30 日關注榜」等大數據應用的資料基石，同時嚴格遵守安全與隱私規範。
- **模組化前端架構**：`data.js`（全域狀態）、`table.js`（渲染）、`ui/`（事件）、`auth/`（驗證）各司其職，易於維護與擴充。
- **Console 零報警**：在 Chrome DevTools Console 與 Issues 面板維持 0 error / 0 warning，確保與瀏覽器政策完全相容。
- **Google OAuth 2.0**：登入流程完全由 Google 伺服器處理，本站不儲存任何用戶密碼。
- **資料安全**：使用者持股清單透過 Supabase Row Level Security (RLS) 保護，只有本人可讀寫。
