# Antigravity (AI Agent) 客製化設定指南

本指南詳細說明如何透過「規則 (Rules)」與「技能 (Skills)」來深度客製化您的 Antigravity 進階代理工程師，使其完美契合您的個人習慣與團隊專案規範。

---

## 區塊一：Rules (規則設定)

`Rules` 是用來規範 Agent 行為邏輯的最基本且最常用的設定。這不只是單純的 Prompt，更是 Agent 在思考與產出程式碼時必須遵守的「最高指導原則」。

設定層級分為兩種：

### 1. Global Rules (全域規則)
*   **作用範圍**：綁定於您的編輯器（如 VS Code）與本機環境。無論您打開哪一個專案資料夾，Agent 都會遵守這些規則。
*   **適用情境**：您的個人開發習慣。
*   **範例**：
    *   「請一律使用繁體中文回答我。」
    *   「請永遠使用 Tab 而非 Space 進行縮排。」
    *   「不要隨意刪除我在程式碼中寫下的中文註解。」

### 2. Workspace Rules (工作區/專案規則)
*   **作用範圍**：僅限於當前開啟的專案資料夾（Working Directory）。
*   **適用情境**：特定的專案架構規範、團隊共同協議、框架版本限制。
*   **優勢**：
    *   **專案隔離**：專案 A 規定只能用 Vanilla CSS，專案 B 規定必須用 TailwindCSS，透過 Workspace Rules 就能明確切分，防止 Agent 寫錯語法。
    *   **團隊共享**：您可以將這些規則（通常以 `.cursorrules`、`rules.md` 或 `.agent/` 目錄下的文件形式存在）納入 Git 版本控制。當新同事 Clone 專案並打開編輯器時，Agent 就會自動套用這些規則，**瞬間與團隊規範同步**。
*   **範例**：
    *   「本專案的前端使用 Vanilla HTML/JS，絕對禁止引入 React 或 Vue。」
    *   「所有的 API 請求必須使用封裝在 `js/api.js` 中的 `fetchData` 函數。」

## 區塊二：Skills (進階技能擴充)

如果您覺得純文字的 `Rules` 不夠用，或者遇到非常複雜、特化的任務，**`Skills`** 就是讓 Agent 「學習新超能力」的進階外掛機制。

`Skills` 不只是給 Agent 讀的文件，它是**一個完整的資料夾結構**，能夠賦予 Agent 執行特定自動化腳本與理解複雜邏輯的能力。

### 為什麼需要 Skills？
1.  **克服字數與上下文限制**：當一項任務涉及極度複雜的業務邏輯（例如公司內部自製的加密金鑰庫、特殊的部署流程），單一的 Rule 文件寫不下也講不清楚。
2.  **賦予工具化能力**：Skill 允許您提供 Python 或 Bash 腳本給 Agent。當 Agent 需要執行該技能時，它可以直接呼叫您準備好的腳本來完成前置作業或驗證結果。

### 典型的 Skill 資料夾結構

一個完整的 Skill 通常會放置在專案的 `.agent/skills/` 或類似的工作區配置目錄下，結構如下：

```text
my-project/
└── .agent/
    └── skills/
        └── 專門處理-Supabase-資料庫同步/    <-- (這是一個 Skill 資料夾)
            ├── SKILL.md                   <-- (必要文件) 技能說明書、運作原理、邊界條件、觸發時機
            ├── scripts/                   <-- 輔助 Agent 在背景執行的腳本 (如 auth_test.py)
            ├── examples/                  <-- 正確的程式碼範例、成功案例模板
            └── resources/                 <-- 該技能需要參考的外部文件或 JSON 配置範本
```

### Skill 運作方式範例
假設您建立了一個名為「Supabase 同步技能」的 Skill：
*   文件中寫明：「當 User 要求備份資料時，請先執行 `scripts/check_env.sh` 確認環境變數，接著參考 `examples/sync_pattern.js` 的寫法來更新資料庫。」
*   當您對 Agent 說：「幫我把最新的股票資料寫入 Supabase」時，Agent 會**自動去翻閱並載入這項 Skill**，完全按照您定義的標準作業程序（SOP）和程式碼風格去執行任務，大幅減少來回溝通與試錯的時間。

---

## 總結

*   用 **Global Rules** 來塑造符合您個人風格的 AI 夥伴。
*   用 **Workspace Rules** 來確保 AI 不會踩到這份專案的雷點（適合推播給團隊使用）。
*   用 **Skills** 來把公司內部複雜的 SOP、自製工具、龐大文件，轉化為 AI 可以直接呼叫的自動化能力。
